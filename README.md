# Pathfinder

Pathfinder は、特定のフォーマットを持たない Web サイトからイベント開催場所の住所を特定・抽出するためのエージェント指向プロジェクトです。LLM を判断エンジン、ブラウザ操作を実行レイヤーとして使い、HTML・画像・PDF を横断して住所候補を探索します。

## 概要
- 不定形な Web サイトから住所を探索することを目的としたプロジェクトです。
- LangGraph を使って探索フローを構成します。
- 現状は LangGraph の最小動作確認用サンプルとして [hello_graph.py](hello_graph.py) を含みます。

## セットアップ
```bash
pip install -r requirements.txt
```

## 実行
```bash
python3 hello_graph.py
```

想定される出力例:

```python
{'message': 'Start -> Hello from Pathfinder!'}
```

## 要件定義

## 1. 目的
特定のフォーマットを持たないWebサイト（PDF、画像、多階層構造を含む）から、イベントの開催場所である「住所」を、人間と同等の探索プロセスで特定・抽出する。

## 2. システム概要
本システムは、LLMを「判断エンジン」とし、ブラウザ操作（Playwright）を「手足」として利用する自律エージェント型システムとする。特定のサイトルールを定義せず、プロンプトベースの推論によって探索先を決定する。

## 3. 処理フロー（エージェントワークフロー）

### 3.1 探索フェーズ（Navigation）
- 初期分析: 指定されたURLをブラウザで読み込み、メインビューのHTML構造とスクリーンショットを解析。
- 優先度付け: リンク一覧およびメニュー項目から、「アクセス」「開催概要」「詳細」「会場」といった住所情報が記載されている可能性が高いリンクを順位付けする。
- 再帰的探索: 特定できなかった場合、優先度の高いリンクを順に辿る（最大深度または探索回数はパラメータで制限する）。

### 3.2 抽出フェーズ（Extraction）
以下の3つのソースに対して、優先順位をつけて抽出を行う。

- テキスト解析: HTMLタグのテキスト内容から、住所形式（郵便番号、県、市、町名、番地）を抽出。
- 画像解析（Vision）: ページ内に埋め込まれた画像またはスクリーンショットを解析し、文字情報を読み取る（OCR代用）。
- ドキュメント解析（PDF）: リンク先がPDFである場合、その内容をテキストまたは画像として解析対象に取り込む。

### 3.3 検証フェーズ（Validation）
- 住所の整合性確認: 抽出された情報が「建物名」「施設名」「住所」として論理的に成立しているかLLMが判定。
- 確実性スコアリング: 抽出結果に対して「High/Medium/Low」の確実性を付与。低確実性の場合は、探索を継続する。

## 4. コンポーネント要件
| コンポーネント | 技術要件 | 役割 |
| --- | --- | --- |
| エージェント制御 | LangGraph (Python) | 探索、解析、終了の意思決定ロジック |
| ブラウザ操作 | Playwright | ページ遷移、レンダリング、スクリーンショット撮影 |
| 推論エンジン | ローカルLLM (Llava/Mistral等) | 状況判断、住所の抽出、意味理解 |
| PDF処理 | marker または PyMuPDF | PDFのテキスト化および画像化 |

## 5. 入出力定義

### 5.1 入力
- `target_url`: 探索を開始するイベントのトップページURL。

### 5.2 出力（JSON形式）
```json
{
  "event_name": "イベント名",
  "address": "抽出された住所",
  "confidence": "High/Medium/Low",
  "source_url": "住所が特定されたURL",
  "reasoning": "住所と判断した根拠",
  "metadata": {
    "is_pdf": boolean,
    "found_in_image": boolean
  }
}
```

## 6. 非機能要件
- 拡張性: サイトのレイアウト変更に影響を受けないよう、ハードコーディングされたXPath等は極力使用しない。
- 安全性: ローカルLLMを使用し、外部へのAPI通信を最小限に抑える（プライバシー保護）。
- ループ制御: 誤ったループによるリソース枯渇を防ぐため、最大遷移数およびタイムアウトを設定する。
